# sage-mcp agents — collector roster

11 dedicated wallets (`deployer` + `collector-1`..`collector-10`) for
collecting artwork on **sageart.xyz (mainnet)**, each with its own
ready-to-paste MCP config. Generated by `generate.js` — private keys and
configs live in `wallets/`, which is **gitignored** (see the root
`.gitignore`); nothing under it is ever committed.

## Generate (or regenerate) the roster

```bash
cd sage-mcp
node agents/generate.js
```

Re-running overwrites `wallets/` with a **fresh** set of 11 wallets — don't
run it again once you've funded the current set, or you'll orphan the funds
on wallets nobody has the new config for.

## What you get

- `wallets/<name>.mcp.json` — one file per agent, each a complete
  `mcpServers` block. Paste the contents straight into
  `claude_desktop_config.json` (merge the inner `"sage-<name>": {...}` entry
  if you already have other MCP servers configured), or use the file's path
  directly for Claude Code.
- `wallets/manifest.json` / `wallets/ADDRESSES.txt` — addresses only (no
  keys), for your own records and for funding.

Each config already points at the **live mainnet** marketplace (sageart.xyz,
chain 4663) — the same contracts the dashboard and real collectors use. No
edits needed before funding + connecting.

## Funding

Every wallet needs its own funds — there's no way to derive a private key
from an address, so nothing here can be funded after the fact without the
address list in `ADDRESSES.txt`. Send **mainnet ETH (Robinhood Chain, chain
id 4663)** to each address you want active:

- Gas for marketplace actions (mint / bid / ticket) is a few thousandths of a
  cent at current network gas prices — a tiny amount covers many
  transactions.
- If an agent should also **buy SAGE** (`sage_buy_sage`), it needs enough ETH
  to cover that swap on top of gas — that amount is your call, not a fixed
  requirement.

Fund only what you're comfortable letting each agent spend autonomously —
per the top-level `sage-mcp/README.md` security notes, these are meant to be
disposable, dedicated wallets, never a treasury key.

## Running them

Each `.mcp.json` is a fully standalone Claude Desktop / Claude Code config —
add one (or several, under different server names) to drive multiple agents
from one client, or hand individual configs to separate machines/instances.
Once connected and funded, a natural first prompt is:

> list SAGE drops, then collect one piece of artwork from a live collection

The agent will call `sage_list_drops` (now includes `collections[]` for
sequential-mint drops like rMonet) and `sage_mint_collection` to actually
collect. See the main README's tool table for everything else available
(open editions, lotteries, auctions, SAGE Social).

## Not built here (yet)

The `deployer` wallet is currently identical to the collectors — sage-mcp
has no drop-deployment tool. Creating a drop requires on-chain `role.admin`
and the dashboard's upload/approve flow, which is a materially bigger
permission grant than "let this wallet buy things." If you want an agent
that can actually deploy drops, that's a separate, explicit decision (admin
role goes to a specific wallet you choose) — ask for it directly rather than
assuming this roster covers it.
