SageStorage: revoke admin wallet's DEFAULT_ADMIN_ROLE

The admin/deployer wallet holds OpenZeppelin's root DEFAULT_ADMIN_ROLE in addition to its intended ADMIN_ROLE — meaning it can currently grant/revoke MINTER_ROLE, BURNER_ROLE, MANAGE_POINTS_ROLE, and DEFAULT_ADMIN_ROLE itself, including stripping the multisig's own privileges. This revokes just that excess role from the admin wallet. It keeps ADMIN_ROLE and everything that gates (SageConfig, SageWhitelist, SageStorage setAddress/deleteAddress, granting ARTIST_ROLE, various onlyAdmin game functions) — it only loses the ability to manage the four DEFAULT_ADMIN_ROLE-gated roles, which become multisig-only going forward. The multisig executing this itself (rather than the admin wallet self-revoking) matches how Auction's own onlyMultisig fix was rolled out.
